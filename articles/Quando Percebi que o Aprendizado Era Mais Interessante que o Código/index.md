# Quando Percebi que o Aprendizado Era Mais Interessante que o Código

![alt text](imagem_1.png)

O Sword & Ledger começou como um projeto simples.

A ideia era construir uma pequena simulação em Python para aprender engenharia de software de forma prática.

Sem frameworks, sem ambições comerciais e sem a preocupação de criar um produto.

Apenas a vontade de aprender construindo algo real.

Durante os primeiros commits, a maior parte da minha atenção estava voltada para o código: novas funcionalidades surgiam, classes apareciam, testes eram adicionados, refatorações aconteciam mas algo curioso começou a acontecer.

Os momentos mais interessantes do projeto não eram necessariamente as funcionalidades implementadas.

Eram as reflexões que surgiam durante sua construção.

Perguntas como:

* Quando uma classe merece existir?
* Como saber se uma abstração está ajudando ou atrapalhando?
* Qual é o momento correto para escrever testes?
* Como evitar overengineering?
* Quando uma refatoração realmente vale a pena?

Comecei a perceber que estava produzindo dois artefatos diferentes.

O primeiro era o próprio Sword & Ledger.

O segundo era um conjunto crescente de aprendizados sobre engenharia de software.

Esses aprendizados apareciam em anotações, conversas, revisões arquiteturais e documentos internos.

Em algum momento ficou claro que eles possuíam valor próprio.

Foi assim que surgiu o Sword & Ledger Journal.

O objetivo deste projeto não é ensinar.

Também não é servir como portfólio.

E não pretende ser uma vitrine de código.

O Journal existe para registrar uma jornada de aprendizado.

Um registro honesto das decisões tomadas, dos erros cometidos, das hipóteses testadas e das lições aprendidas ao longo da construção de um software real.

Talvez algumas dessas reflexões sejam úteis para outras pessoas. Talvez não. Mas isso é secundário.

O objetivo principal é preservar o aprendizado.

Porque projetos terminam.

Tecnologias mudam.

Mas as lições aprendidas durante a jornada costumam permanecer.

E este blog existe para registrá-las.
